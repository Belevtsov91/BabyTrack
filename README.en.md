## 🧬 Origin Story

It all started with one app from the **App Store** — simple but incredibly useful. It genuinely helped during the first two years: every feeding, every nap, every diaper change — all at hand in the middle of the night.

But it had its limitations. And one day the thought came: *what if we take this idea and make it better? Add what was missing, remove what was annoying, and build something of our own.*

On **January 25, 2025** this project was born. And this is not the end — it's just the beginning. The app will keep evolving, improving, and getting new features. Because good ideas don't stop.

---

This project is an honest answer to the question **"what is vibe coding actually like?"**

```
1. 💡 Idea        →  Lovable.dev       (built the UI in one evening, pure magic)
2. 🔧 Development  →  Claude + ChatGPT  (explained, fixed, rewrote)
3. 🚀 Final look   →  Hands + brain     (can't escape this part)
```

**Spoiler:** vibe coding works. Just not the way TikTok shows it.

> *Without a minimum understanding of what's happening in the code — you're not vibe coding,*
> *you're just clicking buttons and praying. That's called something else.*

The difference between a "vibe coder" and "just hoping":
- You understand **what** you're asking to be done, even if you don't know **how**
- You read what AI wrote and understand at least the logic
- You can describe a bug in words, not just "it doesn't work"

**Lovable** gave a beautiful start in hours instead of days.
**Claude** helped with architecture, fixed bugs and kept things from breaking.
**ChatGPT** helped formulate ideas when thoughts hadn't yet become concrete questions.

Result: a project that actually works, not just a screenshot for social media. 🎯

## ✨ What is this

An app for tracking everything that happens with your baby — from the first feeding to the first word. Built with love, caffeine, and a bit of Claude AI at 3am.

No backend. No cloud. Just you, the browser, and **localStorage** as a trusted friend.

## 🎓 Honest take on vibe coding

The biggest lesson from this project was not that AI "writes code for you", but that it very quickly generates direction — but almost never completes product integration without manual engineering work. The most beautiful pieces of code most often broke not at the idea level, but at the junction with a real project: routing, state, responsive design, existing components, local data and already established UI patterns.

Each tool played its role:

> **Lovable** gave a visual scaffold in one evening — something that would have taken days manually.
> **Claude** helped with architecture, debugging and connecting components to real data.
> **ChatGPT** — when a thought hadn't yet formed into a concrete question, helped find the direction.

### Real problems we faced

- AI regularly brought beautiful components but for a different stack: `next/link`, strange tokens, demo data we don't have
- Many blocks looked fine in a screenshot but broke on tablet/mobile — especially sticky sections, onboarding and complex hero blocks
- Logic duplication appeared frequently: new beautiful component on top, old working one below. Then we had to manually remove the second source of truth
- Effects like liquid/glow looked impressive at first but got in the way of UX. Had to cut them to hover/focus/active only
- AI often made mistakes in TypeScript details and architecture: filename conflicts, incompatible props, duplicate UI components, wrong assumptions about data
- Every new session required re-explaining the context: here's our auth-flow, here's how storage works, here's why we don't use X
- AI creates a beautiful abstraction for its own data model, not yours. You end up choosing: reshape your data to fit the component, or reshape the component to fit your data. Both cost time that "vibe coding" supposedly saves

### What actually helped

- Using AI as a fast draft and direction generator, not as the final author
- Immediately checking how a component lives within the existing UX, not evaluating it from an isolated screenshot
- Cutting quickly: if a new block creates duplication — the old version needs to be removed, or the new one not introduced
- After every significant change, running a build and checking real screen sizes

> *In short: vibe coding compresses weeks into days — but not the way TikTok shows.
> They show the first 20%. The remaining 80% is regular engineering work, just with a better draft in hand.*

## 🛠️ Patterns & Skills developed along the way

### React / Architecture

- **`react-hooks/purity`** — `Date.now()` and side effects can't be called in render. All event handlers wrapped in `useCallback`
- **Functional setState** — eliminated stale closures in timers: `setState(prev => ...)` instead of reading state directly in callbacks
- **Removed `AnimatePresence mode="wait"`** from page transitions — exit animation was blocking new page mounting, navigation was freezing
- **TypeScript without `as any`** — typed via `BabyEvent['type']` instead of casts

### UI / Design System

- **`LiquidSurface`** as the single UI primitive for all interactive elements — buttons, inputs, cards
- **`hsl(var(--primary))`** color system via CSS variables — no hardcoded colors in code
- **Category tokens** `--sleep`, `--feeding`, `--diaper` — each event type has its own color from the theme
- **Liquid/glow only on hover/focus/active** — don't keep the interface "glowing" constantly

### Data / Auth

- **localStorage-first architecture** — all data locally, full backend independence
- **Full auth-flow**: registration → login → forgot password → token → session — all via localStorage
- **btoa-based hashing** for passwords until a proper backend exists

### Tools / Process

- **CLAUDE.md** with principal-level architect rules — read-only by default, minimal changes, cross-layer inspection before any decision
- **Memory system** — architectural decisions saved between sessions so AI doesn't forget agreed decisions
- **MCP servers**: Context7 for docs, Playwright for automatic screenshots, n8n for automation
- **VS Code** configured for the project: ESLint + Prettier + Tailwind IntelliSense + Path aliases `@/`
